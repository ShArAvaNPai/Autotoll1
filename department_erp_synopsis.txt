1. SYNOPSIS
1.1 Title of the Project
“Department ERP System”

1.2 Objective of the Project
 To provide a computerized data storage facility. Lecturers can easily search any records of students.
 To reduce paperwork and computational time.
 The application is user-friendly and requires no professional technical skills or knowledge to operate.
 It helps to keep the record of student attendance and marks.
 It helps in managing the lectures timetable.

1.3 Project Category
Web-based Application.

1.4 Software and Hardware requirements
1. Software Requirements
 Operating System : Windows XP and above.
 Front End : HTML 5, CSS, JAVASCRIPT.
 Scripting Language : PHP.
 Back End : MySQL.
 Browser : Mozilla Firefox, Chrome etc.

2. Hardware Requirements
 Processor : Pentium IV 2GHZ and above.
 Memory : Minimum 1GB RAM / 2GB RAM.
 Hard Disk : Minimum Capacity - 150 GB.

1.5 Structure of the program
1.5.1 Analysis:
Our project helps lecturers of the department to maintain the records of student marks and attendance. Without any manual computing, they can get the attendance and mark averages.
Activities: -
 Lecturers can easily maintain student details like attendance and marks information.
 Students can get information of marks and attendance with the help of login.
 Class transfer can be done by the lecturer.
 Admin will maintain staff login and student’s login.
 Students can send feedback.
 Staff can update the events.
